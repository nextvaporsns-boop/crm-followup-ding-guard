# crm_followup_ding_guard

独立项目：每天 `18:00` 读取伙伴云“线索跟进总表”，对当日跟进数小于 `5` 的人员发送钉钉提醒。

当前实现策略：

- 首次提醒：`18:00` 发送工作通知。
- DING：如果配置了 `DINGTALK_DING_ROBOT_CODE`，会同时发送机器人 DING。
- 催办：根据工作通知的 `task_id` 查询已读/未读名单，对未读且仍未达标的人按间隔再次提醒。

## 启动

```bash
pip install -r requirements.txt
python run.py
```

打开 `http://127.0.0.1:8090`。

## 默认调度

- `18:00` 首次检查并发送提醒。
- `18:15` 到 `21:45` 每 15 分钟催办未读且仍未达标的人。

## 说明

- 当前项目已经复用了旧项目的伙伴云表配置和钉钉应用配置。
- 如果暂时没有 `DINGTALK_DING_ROBOT_CODE`，项目仍会正常发送工作通知并根据已读回执做催办。
